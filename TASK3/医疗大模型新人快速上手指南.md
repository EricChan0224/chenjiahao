## 一、工具篇：从零开始把环境跑起来

### 1.1 Python 环境怎么配

不需要从头学 Python 语法，先把环境装好、能跑代码就行：

1. 安装 **Anaconda**（自带 Python + Jupyter Notebook，新手最省心）。
2. 打开 Anaconda Prompt（或终端），建一个干净的虚拟环境，避免和电脑里其他项目的包冲突：
   ```bash
   conda create -n med_ai python=3.10
   conda activate med_ai
   ```
3. 拿到任何一个项目的 `requirements.txt`，一行命令把依赖全部装好：
   ```bash
   pip install -r requirements.txt
   ```

### 1.2 国内怎么调用大模型 API

推荐用**阿里云百炼平台（Qwen 通义千问）**，新用户有免费额度，注册流程大概是：

1. 阿里云官网注册账号 → 开通"百炼"产品 → 创建 API-KEY。
2. 把 API-KEY 存到环境变量里，**不要直接写在代码里发到 GitHub 上**。
3. 用官方提供的 Python SDK 或者 `requests` 库发一个最简单的请求，先确认能跑通，再去写复杂的 Prompt。

### 1.3 怎么用 AI 辅助编程效率最高

几个亲测有用的习惯：

- **别一次性甩需求**：与其一上来就说"帮我写一个完整的心衰预测项目"，不如拆成"先帮我读取数据并做描述性统计"这种一步一步来，每步都能检查结果对不对。
- **报错了就把完整报错信息复制粘贴给 AI**，不要自己口头转述"它说有个错误"——AI 看到原始报错信息（包括具体哪一行、什么类型的 Error）定位问题的速度会快很多。
- **多问"为什么"，少只要"怎么改"**：比如发现两个模型的结果不一样，先让 AI 解释原理，理解之后再决定要不要改，不然容易被表面上"看起来对了"的修改误导。

## 二、实战篇：代码跑不通怎么办

### 2.1 真实案例：Notebook Cell 顺序全乱了

**场景**：拿到的 notebook 从上到下点 Run All，第一个 cell 就报 `NameError: name 'df' is not defined`。

**怎么问 AI 的：**

> 我上传了完整的 notebook 文件，为什么 Cell 10 跑不通？报错是 NameError。不要调换 cell 顺序的情况下能跑通吗？另外有没有办法快速批量调整 cell 顺序？

这个提问的关键是：**附上了完整文件、给了具体报错、明确了想要的两种解法**（不调顺序 / 批量调顺序），而不是只说"报错了帮我看看"。AI 帮我定位到的根因是：notebook 里所有 cell 的物理顺序和它们标注的逻辑顺序刚好完全相反，写代码的人把 cell 整个倒序粘贴了。

> 📌 *详细对话记录见 `AI_Chat_Records.md` —「任务二：Notebook Cell 顺序排查」*

### 2.2 真实案例：模型结果"看起来不对"

**场景**：随机森林的召回率比逻辑回归低，第一反应是"是不是哪里写错了"。

**怎么问 AI 的：**

> 为什么逻辑回归的召回率会比随机森林的召回率高？这个结果合理吗？不合理的话怎么改进？

没有直接默认"这是个 bug 要改"，而是先问"合理吗"——这一步很重要。AI 解释完线性模型 vs 集成模型的机制差异之后，我才确认这其实是正常现象，而不是代码错误，避免了瞎改一通把本来正确的结果改坏。

> 📌 *详细对话记录见 `AI_Chat_Records.md` —「任务二：模型召回率差异排查」*

### 2.3 几个最容易踩的坑

- **变量名撞车**：两段代码里用了同一个变量名（比如都叫 `model`），后面的会把前面的悄悄覆盖掉，结果错得莫名其妙还不报错，最难排查。
- **没固定随机种子**：每次重新跑代码结果都不一样，论文里写的数字根本对不上。一定要在代码开头统一设一个 `random_state`，所有用到随机性的地方（切分数据集、训练模型）都用同一个值。
- **用了"不该用"的特征**：比如本例中的 `time`（随访时长），这种在预测当下根本不存在的信息如果被喂给模型，会让结果好得不真实——这叫**数据泄漏**，是医疗 AI 项目里最容易被审稿人挑出来的硬伤。

### 2.4 一个通用的高效 Debug 提问公式

```
【报错信息】完整粘贴报错文本（不要自己转述）
【我做了什么】我执行了哪一步操作
【我期望什么】我原本期望看到什么结果
【附加信息】相关代码文件 / 截图
```

按这个公式提问，AI 基本不会答非所问。

## 三、进阶篇：AI 辅助科研绘图

### 3.1 怎么让 AI 画图更"对味"

直接说"帮我画一个流程图"，AI 给的图大概率很普通。更有效的做法是：

1. **先用文字把图的内容结构列清楚**，比如"五个阶段：数据输入 → 预处理 → 建模 → 评估 → 输出，每个阶段下面再列 2-3 个要点"，确认这份大纲没问题，再让 AI 生成图片。
2. **明确给出风格关键词**：比如"科技感、医疗严谨感、配色偏蓝绿色调、扁平化图标风格"，比泛泛地说"好看一点"有用得多。
3. 一次给的信息密度不要太大，**专业术语、英文缩写要写准确**（比如 `class_weight`、AUC 这种），AI 生图工具经常会把不熟悉的术语拼错。

### 3.2 怎么缓解 AI 生图的"幻觉"问题

AI 生成的图里**最容易出错的是文字和数字**（拼写错误、数值编造），图形结构本身相对可靠。应对方法：

- **生图前先核对内容大纲，生图后再核对一遍图里的文字**，不要图好看就直接用，尤其是论文里要正式引用的图。
- **关键数字不要让 AI 在生图阶段"顺手编"**，应该是先有你自己代码跑出来的真实数字，再明确告诉 AI"图里这几个数字必须是 0.7333、0.746 这种"，而不是让它自由发挥。
- 如果图里文字出现乱码或者错别字，**直接让 AI 重新生成那部分**，不要自己用画图软件手动抠图改——容易越改越乱，不如让 AI 重做一版。

## 写在最后

这几个挑战做下来最大的感受是：AI 不是用来"代劳"的，而是用来**加速你自己的理解**的。报错了先弄懂原因再改，结果反常先问"为什么"再决定要不要动——这个习惯比记住多少个 Prompt 模板都重要。希望这份指南能帮你少走一点弯路。
